CREATING A NEW MICROSERVICE IN BOOK-MY-MOVIE

1. Create Service Directory

From the project root:

book-my-movie> mkdir <service-name>

book-my-movie> cd <service-name>

Example:

book-my-movie> mkdir auth

book-my-movie> cd auth

---

2. Initialize Node.js Project

npm init -y

---

3. Install Required Dependencies

Install project-specific dependencies.

Example:

npm install express cors

npm install -D typescript ts-node-dev @types/node @types/express

---

4. Create Application Entry Point

Create:

src/
└── index.ts

Initialize Express server and verify application startup.

---

5. Configure Package Scripts

Add scripts in package.json:

"scripts": {
"start": "ts-node-dev src/index.ts"
}

Run service:

npm run start

---

6. Create Docker Image

Build Docker image from the service directory:

docker build -t <docker-id>/<image-name> .

Example:

docker build -t adarsh893/auth .

---

7. Push Docker Image

Push image to Docker Hub:

docker push <docker-id>/<image-name>

Example:

docker push adarsh893/auth

---

8. Configure Skaffold

Add the new image in skaffold.yaml:

build:
artifacts:
- image: adarsh893/auth

This enables automatic image rebuilds and deployments during development.

---

9. Create Kubernetes Deployment

Create deployment YAML:

infra/k8s/auth-depl.yaml

Deployment Responsibilities:

* Runs service containers
* Maintains desired replica count
* Handles pod restarts
* Supports rolling updates

---

10. Create ClusterIP Service

Expose deployment internally:

auth-cluster-ip-srv

Responsibilities:

* Internal service discovery
* Stable networking endpoint
* Communication between microservices

---

11. Create Database Resources (If Required)

Each microservice should own its database.

PostgreSQL:

auth-postgres-depl

auth-postgres-cluster-ip-srv

MongoDB:

auth-mongo-depl

auth-mongo-cluster-ip-srv

---

12. Start Development Environment

From project root:

book-my-movie> skaffold dev

Skaffold will:

* Watch source code changes
* Rebuild Docker images
* Push updated images
* Redeploy Kubernetes pods automatically

---

IMPORTANT NOTES

Skaffold Usage:

Skaffold is intended only for development environments.

Do NOT use Skaffold in production.

Production deployments typically use:

* GitHub Actions
* Jenkins
* ArgoCD
* FluxCD
* GitLab CI/CD

---

Live Reloading

Skaffold rebuilds and redeploys containers but does not automatically restart the Node.js application.

Use:

* ts-node-dev
  or
* nodemon

Example:

"scripts": {
"start": "ts-node-dev src/index.ts"
}

This enables automatic application restarts when source files change.

---

Development Workflow Summary

1. Create Microservice
2. Initialize Node Project
3. Install Dependencies
4. Create Express Server
5. Build Docker Image
6. Push Docker Image
7. Configure Skaffold
8. Create Deployment
9. Create ClusterIP Service
10. Create Database Resources (if required)
11. Run "skaffold dev"
12. Start Development
